DataScout 摘要撰寫要點：

背景陳述 (100-150字)：
隨著大數據時代來臨，資料科學已成為推動數位轉型的核心動力。然而，傳統資料科學專案面臨工具分散、流程複雜、重複性工作繁重等挑戰。現有解決方案往往缺乏統一的自適應機制，難以應對多變的業務需求和技術環境。

## 📚 學術寫作技巧提示詞

### **學術語言與表達**
```
學術寫作語言規範：

客觀性表達：
- 使用第三人稱：「本研究」、「該系統」
- 避免主觀判斷：「明顯地」→「根據數據顯示」
- 謹慎用詞：「可能」、「似乎」→「結果表明」

精確性要求：
- 量化描述：「大幅提升」→「提升95%」
- 具體數據：「很快」→「287毫秒內」
- 明確範圍：「通常」→「在95%的情況下」

邏輯連接：
- 因果關係：「因此」、「由於」、「導致」
- 對比關係：「相較之下」、「然而」、「反之」
- 遞進關係：「此外」、「進一步」、「更重要的是」

專業術語：
- 一致性使用：首次出現時定義，之後統一使用
- 縮寫規範：Application Programming Interface (API)
- 技術名詞：保持英文原文或提供對照
```

### **圖表設計與說明**
```
學術圖表規範：

圖表編號：
- 圖：Figure 1、Figure 2...
- 表：Table 1、Table 2...
- 公式：Equation (1)、Equation (2)...

圖表標題：
- 位置：圖下表上
- 內容：完整描述圖表內容
- 格式：Figure 1. DataScout系統架構圖

圖表內容：
- 清晰度：至少300dpi解析度
- 字體：與正文一致，最小10pt
- 色彩：考慮黑白印刷效果
- 標籤：軸標籤、圖例清楚標示

引用格式：
- 正文中引用：「如圖1所示」、「見表2」
- 每個圖表都必須在正文中提及
- 先在正文中討論，再呈現圖表
```

### **參考文獻管理**
```
引用格式規範（APA格式）：

期刊論文：
Author, A. A. (Year). Title of article. Title of Journal, Volume(Issue), pages. DOI

會議論文：
Author, A. A. (Year). Title of paper. In Title of Conference Proceedings (pp. pages). Publisher.

書籍：
Author, A. A. (Year). Title of book. Publisher.

網站資源：
Author, A. A. (Year, Month Date). Title of web page. Website Name. URL

軟體工具：
Developer. (Year). Software Name (Version) [Computer software]. URL

正文引用：
- 單一作者：(Smith, 2023)
- 多位作者：(Smith & Jones, 2023)
- 三位以上：(Smith et al., 2023)
- 多篇文獻：(Smith, 2023; Jones, 2022)

引用管理：
- 使用 Zotero 或 Mendeley 管理文獻
- 保持格式一致性
- 確保所有引用都有對應的參考文獻
```
